## Features

  variant="list"
  data={[
    <Typography key="semantic-html">

      <Typography variant="inlineCode">h1</Typography>,{" "}
      <Typography variant="inlineCode">p</Typography>,{" "}
      <Typography variant="inlineCode">blockquote</Typography>,{" "}
      <Typography variant="inlineCode">code</Typography>) based on the selected
      variant.
    </Typography>,
    <Typography key="responsive-sizing">
      <Typography variant="strong">Responsive Sizing:</Typography> Heading
      variants come with pre-configured fluid typography (e.g.,{" "}
      <Typography variant="inlineCode">h1</Typography> transitions between sizes
      on mobile/desktop).
    </Typography>,
    <Typography key="data-driven">
      <Typography variant="strong">Data Driven Lists:</Typography> The `list`
      variant accepts a `data` array prop for cleaner JSX when rendering bullet
      points.
    </Typography>,
    <Typography key="consistent-spacing">
      <Typography variant="strong">Consistent Spacing:</Typography> Built-in
      margin handling for standard document flow (e.g., `:not(:first-child)`
      logic for paragraphs and headers).
    </Typography>,
  ]}
/>

## Props

| Prop      | Type           | Default | Description                                      |
| --------- | -------------- | ------- | ------------------------------------------------ |
| `variant` | `string`       | `"p"`   | Visual style and semantic mapping.               |
| `as`      | `ElementType`  | -       | Manual override for the underlying HTML element. |
| `data`    | `React.Node[]` | -       | Array of items to render when `variant="list"`.  |

<Typography variant="blockquote">
  Accepts all standard HTML attributes for the underlying element.
</Typography>
